# junyizhu-ai.github.io

Personal academic homepage of Junyi Zhu. Static site, generated from data files by a
zero-dependency Node script and deployed to GitHub Pages via GitHub Actions.

## How it works

```
_data/papers.bib   ← publications (BibTeX). Edit this to add/update papers.
_data/news.yml     ← news items (date + one line of HTML).
_data/bio.html     ← homepage bio, verbatim HTML paragraphs.
build.mjs          ← generator: data → index.html, publications/, paper pages, sitemap.xml
assets/            ← css, images (img/pub/*.jpg are web-optimized thumbnails)
robots.txt         ← allows all crawlers (incl. GPTBot, ClaudeBot, PerplexityBot)
llms.txt           ← generated by build.mjs — who I am + site structure, for AI crawlers
.github/workflows/deploy.yml ← runs `node build.mjs`, deploys to Pages
```

Generated files (`index.html`, `publications/**`, `sitemap.xml`, `404.html`) are rebuilt
on every push — edit the `_data/` files, never the generated HTML.

## Updating content

- **New paper**: append a BibTeX entry to `_data/papers.bib`. Optional fields:
  `selected={true}` shows it on the homepage; adding `slug={...}` (plus `arxiv`, `code`,
  `abstract`, `summary`) creates a dedicated SEO page at `/publications/<slug>/` with
  ScholarlyArticle JSON-LD. Put a preview image in `assets/img/publication_preview/` and
  add a web-sized copy at `assets/img/pub/<name>.jpg` (~320px wide).
- **News**: add an item at the top of `_data/news.yml`.
- **Bio**: edit `_data/bio.html`.
- Preview locally: `node build.mjs`, then open `index.html`.

## Deploying (first time)

1. Replace the contents of the `JunyiZhu-AI.github.io` repo with these files.
2. In repo Settings → Pages, set **Source: GitHub Actions**.
3. Push — the workflow builds and deploys.

## Notes

- Email address never appears in the HTML source; it is assembled client-side
  (`.js-email` links in `build.mjs`).
- SEO/GEO: Person JSON-LD on the homepage, ScholarlyArticle JSON-LD + unique
  title/description per paper page, sitemap.xml, llms.txt, no crawler restrictions.
- The `Homepage Directions.dc.html` file and `support.js` in the design project are
  exploration artifacts — do not copy them into the repo.
